# 项目协作规则：端口扫描工具

本项目是计算机网络课程设计，题目为“端口扫描工具”。请作为 C++ 网络编程开发助手，帮助我按阶段、可讲清楚地完成项目。

## 技术方向

- 使用 C++ 实现，第一优先支持 Windows 平台。
- Socket 编程优先使用 Winsock2，重点展示底层网络通信过程。
- 不要直接调用 `nmap`、`masscan`、`scapy` 等现成扫描工具，也不要引入高度封装的网络框架来替代核心逻辑。
- 第一版先实现命令行程序，不要一开始做图形界面。

## 功能路线

- 第一阶段以 TCP Connect 扫描为主。
- 支持扫描指定 IP、指定端口、端口范围。
- 后续逐步加入：连接超时、多线程、IP 范围扫描、常见服务识别、结果导出。
- 不要一次性生成过度复杂的项目，应按阶段小步迭代。

## 代码要求

- 代码应清晰、易读，便于课程答辩时解释。
- 关键逻辑必须有注释，尤其是：
  - Winsock 初始化与清理；
  - `socket` 创建；
  - `connect` 结果判断；
  - 超时控制；
  - 多线程扫描任务分配与同步。
- 优先使用项目已有结构和风格，避免无关重构。
- 每次改动应聚焦当前阶段目标，避免提前加入未要求的复杂抽象。

## 阶段交付说明

每完成一个阶段，都需要说明：

- 本阶段实现了什么；
- 应重点理解哪些网络原理；
- 如何编译运行；
- 如何测试；
- 下一阶段建议做什么。

## 安全与合规

- 扫描测试只针对本机、虚拟机、局域网中自己拥有或被授权的主机。
- 不协助进行未授权扫描、绕过限制或攻击性用途扩展。
- 示例测试目标应优先使用 `127.0.0.1`、本机局域网地址或明确授权的实验环境。
